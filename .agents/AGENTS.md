# Fast MatMul Project Instructions for AI Agents

This document outlines the architectural constraints, linking requirements,
compilation flags, and style guidelines that must be strictly followed when
working on the `fast-matmul` codebase.

---

## 1. Matrix Multiplication & Dynamic Peeling

- **No Padding**: We implement **dynamic peeling** mathematically matching the
  definition in the revised `report/report.tex` to handle odd or
  non-power-of-two dimensions. Do not use zero-padding.
- **Modular Implementation**: Keep peeling logic isolated in modular helper
  functions inside
  [matmul.rs](file:///home/alberto/Data/pisa/fast-matmul/rust/src/matmul.rs)
  (e.g. `peel_core`, `correct_inner_dimension`, `correct_right_columns`,
  `correct_bottom_rows`, and the main orchestrator `dynamic_peeling`).

## 2. Intel MKL Linking & Threading Layers

- **Threading Library Choice**: We link dynamically against the GNU OpenMP
  threading layer (`mkl_gnu_thread`, `-lgomp`) rather than the sequential MKL
  library to allow benchmarking both modes in a single executable.
- **C FFI Wrappers**: Due to calling convention and naming mismatches between C
  and Fortran interfaces inside Intel MKL, **never** call `mkl_set_num_threads`
  or `mkl_get_max_threads` directly from Rust. Always call them through the C
  wrappers defined in
  [mkl.c](file:///home/alberto/Data/pisa/fast-matmul/rust/ffi/matmul/mkl.c):
  - `mkl_set_num_threads_wrapper(nt)`
  - `mkl_get_max_threads_wrapper()`
- **Thread Count Management**:
  - Setting threads to `1` runs sequentially.
  - Setting threads to `0` restores parallel execution using all available
    threads.
  - To avoid losing track of the host default CPU cores limit after sequential
    execution has been selected, cache the startup maximum threads count using
    `std::sync::OnceLock` in
    [mkl.rs](file:///home/alberto/Data/pisa/fast-matmul/rust/src/mkl.rs) on the
    first invocation.

## 3. SIMD Target Compilation

- **Rust (LLVM) Optimization**: All cargo compilations must use native target
  features to enable SIMD (AVX2, AVX-512, FMA). Ensure
  [.cargo/config.toml](file:///home/alberto/Data/pisa/fast-matmul/rust/.cargo/config.toml)
  contains `rustflags = ["-C", "target-cpu=native"]`.
- **C Wrapper Optimization**: Ensure the C compiler flags in
  [build.rs](file:///home/alberto/Data/pisa/fast-matmul/rust/build.rs) contain
  target architecture specifications (`-march=native`, `-mtune=native`).

## 4. Benchmarking & Visualization

- **Criterion Harness**: Maintain dual sequential and parallel MKL benchmarks
  (`"MKL-Sequential"` and `"MKL-Parallel"`) in both the programmatic runner
  ([benchmark.rs](file:///home/alberto/Data/pisa/fast-matmul/rust/src/benchmark.rs))
  and the benchmark harness
  ([matmul_bench.rs](file:///home/alberto/Data/pisa/fast-matmul/rust/benches/matmul_bench.rs)).
- **CSV Export**: Benchmark results are exported to
  `rust/generated/benchmark_results.csv`.
- **Plot Generation**: Performance plots are generated by
  [plot.py](file:///home/alberto/Data/pisa/fast-matmul/python/plot.py) using
  `uv run python/plot.py`. When adding new algorithms, explicitly define their
  line styles, markers, and colors in the `styles` dictionary inside `plot.py`
  to maintain a professional, cohesive color palette.

## 5. Development Workflow

- Always run `cargo check --all-targets` and `cargo test` before submitting
  changes.
- Validate that your changes compile without warnings under
  `cargo clippy --all-targets`.
- Write clean code, according to skills/clean-code.
